About <meta>

The <meta> tag defines the metadata of an HTML document. Metadata is data that describes data!

<meta> tags go inside the <head> element, and are usually used to specify the character set, page description, keywords, author(s) of the document, and viewport settings (the user's visible area of a webpage). Multiple Meta elements with different attributes can be used on the same webpage.

While metadata won't be displayed on the webpage itself, it is used by browsers (to display content or reload page), search engines (keywords), and other web-based services.

<meta> Attributes

Attribute Value Description
charset character_set Specifies the character encoding for the HTML document
content text Specifies the value associated with the http-equiv or name attribute
http-equiv content-secruity-policy, content-type, default-style, refresh Provies an HTTP header for the information/value of the content attribute
name application-name, author, description, generator, keywords, viewport Specifies a name for the metadata

charset Attribute Values

charset Value Syntax Example Description
utf-8 <meta charset="utf-8"> The Unicode Consortium develops the Unicode Standard. UTF-8 is a universal character set that includes almost any character from any human language. Your webpage will be able to display any language. Like, Chinese: 汉字 or 漢字. The default character encoding in HTML-5 is UTF-8.
utf-16
ISO-8859-1 <meta charset="ISO-8859-1" The International Standards Organization is the character set for the Latin alphabet which covers North America, Western Europe, Latin America, the Carribean, and Africa. The webpage won't be able to render languages like Chinese or Hindi (which are the 2nd and 3rd most spoken languages worldwide respectively, with English ranking in first). There are different variants of ISO-8859.
ASCII <meta charset="ascii"> ASCII stands for the American Standard Code for Information Interchange. It was created to standardize the storing of text by defining a unique binary number for each storable character to support upper and lower case alphabet, numbers from 0-9, and special characters. Limited to represent 128 different characters, and excludes non-English letters. Mainly in use with large mainframe computer systems.
windows-1252 Windows-1252 was the first default character set in Microsoft Windows. It is an extended version of ASCII with additional international characters. It uses a full byte (8-bits) to represent 256 different characters. It is supported by all browser

http-equiv Attribute Values

The http-equiv attribute provides an HTTP header for the information/value of the content attribute. The http-equiv attribute can be used to simulate an HTTP response header.

http-equiv Value Syntax Example Description
content-security-policy <meta http-equiv="content-security-policy" content="default-src 'self'"> Specifies a content policy for the document.
content-type <meta http-equiv="content-type" conent="text/html; charset=UTF-8"> Specifies the character encoding for the document.
default-style <meta http-equiv="default-style" content="the document's preferred stylesheet"> Specifies the preferred style sheet to use. The value of the content attributes must match the value of the title attribute on a link element in the same document or it must match the value of the title attribute on a style element in the same document.
refresh <meta http-equiv="refresh" content="300"> Defines a time interval for the document to refresh itself. This value should be used carefully as it takes control of a page away from the user. Using refresh will cause a failure in W3C's Accessibility Guidelines.

name Attribute Values

The name attribute specifies the name for the metadata. The name attribute specifies a name for the information/value of the content attribute.

If the http-equiv attribute is set, the name attribute should not be set. HTML5 introduced a method to let web designers take control over the viewport (the user's visible area of a web page), through the <meta> tag (See "Setting The Viewport" example below).

name Value Syntax Example Description
application-name <meta name="application-name" content="webpage name"> Specifies the name of the web application that the page represents
author <meta name="author" content="Binh Lee"> Specifies the name of the author of the document.
description <meta name="description" content="free web tutorials" Specifies a description of the page. Search engines can pick up this description to show with the results of searches.
generator <meta name="generator" content="FrontPage 4.0"> Specifies one of the software packages used to generate the document
keywords <meta name="keywords" content="HTML, meta tag, tag reference"> Specifies a comma-separated list of keywords - relatvant to the page and informs search engines what the page is about.
viewport <meta name="viewport" content="width=device-width, initial-scale=1.0"> Controls the viewport. The viewport varies with the device (smaller on a mobile phone vs a computer screen). A <meta> viewport element tells the browser how to control the page's dimensions & scaling. The "width=device-width" sets the width of the page to follow the screen-width of the device. The "initial-scale=1.0" sets the initial zoom level when the page is first loaded by the browser.
